# 整体验证系统架构设计（草稿）

## 目标
统一整理以下三部分：
1. 验证执行主线：Task / Run / Step / Artifact / Verdict
2. 对话记忆主线：Thread / Message / Summary / Memory / Snapshot
3. 上下文编排层：恢复、压缩、长期记忆提取、任务关联

## 待确认的关键决策
- 存储：SQLite 单库优先 vs PostgreSQL 可迁移结构
- 记忆重心：聊天助手型 vs 项目协作型
- 线程关联：是否允许一个 Thread 关联多个 Task / Run

## 统一边界原则
- 聊天记忆不直接塞进任务表
- 摘要单独建模
- 长期记忆与线程上下文分离
- 任务事实与对话推理分离
- 恢复上下文依赖 snapshot + summary + recent messages

## 下一版输出内容
- 总体架构图
- 模块边界
- 表结构分组
- 上下文恢复链路
- MVP 与二期扩展边界

## 已确认决策
- 存储策略：SQLite 单库优先，表结构按可迁移思路设计
- 记忆重心：项目协作型优先
- 关联模型：支持一个 Thread 关联多个 Task / Run

## 设计落点
- 数据层优先围绕本地桌面单机一致性、事务简单性、可备份性展开
- 记忆系统优先沉淀项目决策、约束、结论、讨论上下文，而不是泛用户闲聊偏好
- Thread 与 Task / Run 采用多对多关联，支持一个讨论线程推进多个验证任务与多次执行
