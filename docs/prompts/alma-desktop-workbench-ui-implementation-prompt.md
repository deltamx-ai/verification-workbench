# Alma 桌面工作台 UI 实现 Prompt

你要实现一个桌面端 AI 工作台页面，整体风格参考深色 macOS 应用，界面是一个三栏布局的会话/任务执行工作区。请直接产出可运行代码，不要只写设计说明。

## 目标
实现一个类似 AI agent 工作台的主界面，重点是：
- 左侧：导航与分类筛选
- 中间：会话列表
- 右侧：当前会话详情 + 执行区
- 深色主题
- 适合桌面端
- UI 要精致，接近 production 级别，而不是 demo 拼凑感

## 页面结构

### 1. 最左侧边栏（导航栏）
宽度较窄，深色背景，包含：
- 顶部一个醒目的「新建会话」按钮
- 会话分类入口
  - 所有会话
  - 积压
  - 待办
  - 待审查
  - 完成
  - 已取消
  - 已标记
  - 已归档
- 数据源分组
  - API
  - MCP
  - 本地文件夹
- 技能分组
- 自动化分组
  - 定时任务
  - 事件触发
  - 智能体
- 底部系统入口
  - 设置
  - 最新动态

要求：
- 每个条目有图标
- 当前选中项有高亮态
- 分组之间有明显层级和留白
- 整体像专业桌面客户端，不像网页后台模板

### 2. 中间栏（会话列表）
用于展示会话/任务流列表，包含：
- 顶部标题：例如“所有会话”
- 顶部右侧有筛选/排序按钮
- 列表按“今天”等时间分组
- 每个会话项展示：
  - 标题
  - 状态指示（圆点/图标）
  - 执行耗时或时间
- 当前选中的会话项要有明显高亮边框或侧边标识
- 列表风格简洁，适合高密度浏览

示例会话：
- Install Codex Claude Skills
- Code Clarity Maintenance

### 3. 右侧主内容区（会话详情）
右侧是当前会话的执行详情页，包含：

#### 顶部
- 当前任务标题
- 右上角操作按钮（如分享、关闭、展开等）

#### 内容主体
展示一张 AI 生成的执行计划卡片：
- 卡片标题：Plan
- 计划内容使用 Markdown 风格排版
- 包含一个任务标题，例如“安装 Claude / Codex Skills 到 Pi”
- 下方是编号步骤列表
- 计划内容区域要有卡片感、边框、圆角、弱阴影
- 深色背景下保持高可读性

#### 底部操作区
在详情内容下方需要一个执行控制区域，包含：
- 主按钮：“执行”
- 次按钮/下拉：“待办”
- 一段状态文本，例如“思考中... 2s”
- 输入框，支持多行输入，placeholder 类似“按 Shift + Return 换行”
- 左下角附件/工具按钮
- 右下角模型选择器，例如“GPT-5.5”

## 交互要求
请实现基础交互，不要只是静态图：
- 点击左侧分类，切换高亮状态
- 点击中间会话列表项，右侧内容切换
- 执行按钮有 hover / active 状态
- 输入框支持聚焦态
- 会话状态支持不同颜色标识
- 结构上为后续接入真实数据预留接口

## 技术要求
- 使用 React + TypeScript
- 样式使用 Tailwind CSS（或你认为更适合的现代方案，但要统一）
- 组件拆分清晰
- 不要依赖重型 UI 库堆砌出效果，尽量自己实现界面质感
- 页面要响应式，但优先优化桌面端
- 代码要可维护，组件命名清晰
- 先给完整代码，再补充简短说明

## 视觉风格要求
- 整体是高级深色主题
- 圆角偏大，接近 macOS app 风格
- 低饱和暗色背景 + 紫色高亮点缀
- 分栏之间层次清晰
- 文字对比度舒服，不刺眼
- 要有“AI 工作台 / agent 控制台”的感觉

## 数据结构建议
你可以先内置 mock data，至少包括：
- sidebar sections
- conversation list
- selected conversation
- plan markdown content
- execution status
- model selector options

## 输出要求
直接输出：
1. 完整页面组件代码
2. 必要的子组件代码
3. mock data
4. 样式实现
5. 如有必要，补一个 `App.tsx` 示例入口

不要只给线框描述，不要只给伪代码，我要的是可以直接开始跑的实现。

## 可选强化要求（高保真模式）
如果模型有余力，请尽量贴近高保真桌面端产品界面，而不是普通管理后台：
- 更细致的间距节奏
- 更讲究的暗色层级
- 更像原生桌面 App 的按钮、列表、卡片质感
- 保留后续扩展为真实 agent 控制台的可能性
